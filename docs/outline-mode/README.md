# 框架约束模式 (Outline-Constrained Mode)

> PRD-11 | 版本 1.0 | 2024

## 什么是框架约束模式？

框架约束模式是写作系统的第 4 种写作模式，专为**有固定框架要求**的文档设计。

**核心特点**：
- 框架由用户提供，严格固定不可修改
- AI 按框架逐章生成内容
- 支持模板复用
- 自动进行一致性检查

## 适用场景

| 场景 | 典型文档 |
|-----|---------|
| 领导给提纲 | 按提纲补充内容 |
| 强框架约束 | 项目立项报告、开题报告、标书、专利申请 |
| 弱框架约束 | 客户指定要点，基于要点拟制方案 |
| 模板复用 | 需要重复使用固定格式的文档 |

## 与其他模式对比

| 维度 | 教练模式 | 快速模式 | 混合模式 | **框架约束模式** |
|------|---------|---------|---------|-----------------|
| AI生成比例 | 0% | 100% | 40% | 60-80% |
| 框架来源 | AI建议 | AI生成 | AI生成 | **用户提供** |
| 框架可调 | 可调 | 可调 | 可调 | **严格固定** |
| 典型场景 | 个人经历文 | 信息整理 | 教程类 | 强约束文档 |
| AI检测率 | < 15% | 25-40% | 18-25% | 20-35% |

## 快速开始

### 方式 1：直接粘贴提纲

```
/outline

然后粘贴你的提纲文本
```

### 方式 2：使用预置模板

```
/outline --list              # 查看可用模板
/outline --template project-proposal  # 使用项目立项报告模板
```

### 方式 3：通过 /write 命令

```
/write

选择 4. 框架约束模式
```

## 核心概念

### 章节类型

- **通用章节**：背景介绍、研究现状、进度安排等，AI 可自动生成
- **核心章节**：研究内容、技术方案、经费预算等，需要用户提供关键信息

### 要素池

系统会维护一个"要素池"，确保全文一致性：
- 专有名词（项目名称、机构名称）
- 术语定义（AI/人工智能）
- 数据口径（金额、时间）

### 一致性检查

生成完成后，系统会自动检查：
- 项目名称是否统一
- 术语使用是否一致
- 数据是否前后呼应

## 常见问题

**Q: 框架约束模式和混合模式有什么区别？**

A: 核心区别在于框架的来源：
- 混合模式：框架由 AI 生成，可以调整
- 框架约束模式：框架由用户提供，严格固定

**Q: 可以自定义模板吗？**

A: 可以，使用 `/outline --save-template <名称>` 保存当前提纲为模板。

**Q: AI 生成的内容质量如何？**

A: 通用章节 AI 可以较好完成，核心章节需要用户提供关键信息才能生成高质量内容。
